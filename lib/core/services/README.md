# core/services/

Обёртки над внешними сервисами (Firebase, API и т.д.).

| Файл | Описание |
|------|----------|
| `collection_names.dart` | Названия коллекций Firestore (чтобы не писать строки руками) |
| `firebase_auth_service.dart` | Обёртка над `FirebaseAuth` + `GoogleSignIn` |
| `firestore_service.dart` | Обёртка над `FirebaseFirestore` (CRUD-операции) |

## Зачем обёртки?

1. **Изоляция** — если Firebase заменится другим сервисом, меняем только сервис.
2. **Логирование** — все вызовы логируются в одном месте.
3. **Тестирование** — проще подменить один класс при тестировании.

> Сервис знает только про **свой** источник данных.
> Координация нескольких сервисов — задача **репозитория** (`data/repositories/`).
